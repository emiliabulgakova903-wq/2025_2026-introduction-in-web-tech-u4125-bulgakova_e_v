\### Лабораторная работа №1: Работа с Docker и контейнеризация статического сайта



\#### Цель работы:

Ознакомиться с основами работы в Docker, научиться создавать собственные Docker-образы и запускать контейнеры на базе веб-сервера Nginx.



\#### Ход работы:

1\. \*\*Подготовка окружения\*\*: Была выполнена настройка WSL и установлен Docker Desktop.

2\. \*\*Создание структуры проекта\*\*: В репозитории создана отдельная ветка `lab1` и папка для файлов лабораторной работы.

3\. \*\*Написание Dockerfile\*\*: Использован базовый образ `nginx:alpine` для минимизации размера контейнера.

4\. \*\*Настройка контента\*\*: Создан файл `index.html` с корректной кодировкой UTF-8 для отображения кириллицы.

5\. \*\*Сборка и запуск\*\*: 

&nbsp;  - Сборка образа: `docker build -t emilia-web-site ./lab1`

&nbsp;  - Запуск контейнера: `docker run -d -p 8080:80 --name my-lab1-container emilia-web-site`



\#### Результат:

Сайт успешно запущен внутри изолированного контейнера и доступен по адресу `localhost:8080`.



\#### Выводы:

Docker значительно упрощает процесс развертывания приложений, позволяя упаковать всё необходимое (код, сервер, настройки) в один переносимый образ.
